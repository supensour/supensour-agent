# Vue — unit test conventions

Extends `rules/vue/index.md`. Applies to `--type unit` (the default).

## Watermark placement

Text from `scripts/watermark.sh` (default shown).

- **New spec file** → watermark header comment as the first line (see Structure below).
- **New `it()`/`test()` added to an EXISTING spec not created by this skill** → a comment line with the
  watermark directly above each added test; leave the rest of the file untouched:
  ```ts
  // Generated with skill supensour:create-tests · suprayan@supensour · github.com/supensour/supensour-agent
  it('rejects empty input', () => { ... })
  ```
  (A spec is "skill-created" if it already contains `supensour:create-tests`. If so, just add the
  test with no per-test comment.)

## Structure

Every spec nests three levels: **file** → **member** (prefixed by kind) → **scenario** (`it`). Watermark
header comment is the first line, then imports, then the outer `describe` named after the source file,
`<basename>.<ext>`. Components and composables both expose multiple kinds of member (prop/event/computed
for a component; computed/function/watch for a composable) that can collide on name, so both prefix the
member `describe` with its kind. Plain utility modules (just exported functions, nothing reactive) are
the one exception — no prefix, since there's only one kind of member.

New spec file — component:

```ts
// Generated by supensour:create-tests · suprayan@supensour · github.com/supensour/supensour-agent
import { describe, it, expect, vi, beforeEach } from 'vitest'   // omit if globals: true
import { shallowMount, flushPromises } from '@vue/test-utils'
import { createTestingPinia } from '@pinia/testing'
import Component from '@/components/.../Component.vue'

describe('Component.vue', () => {
  beforeEach(() => {
    vi.clearAllMocks()
    // vi.resetModules() when a test re-imports with different mocks
  })

  describe('prop#items', () => {
    it('renders total when items is non-empty', () => {
      const wrapper = shallowMount(Component, {
        props: { /* ... */ },
        global: { plugins: [createTestingPinia({ createSpy: vi.fn })] },
      })
      expect(wrapper.text()).toContain('...')
    })
  })

  describe('event#submit', () => {
    it('emits submit with form payload when submit button is clicked', async () => {
      const wrapper = shallowMount(Component, { props: { /* ... */ } })
      await wrapper.find('[data-test="submit"]').trigger('click')
      expect(wrapper.emitted('submit')?.[0]).toEqual([{ /* payload */ }])
    })
  })
})
```

## Structure — composables & utility modules

```ts
// composable: src/composables/useOrderList.js
describe('useOrderList.js', () => {
  describe('computed#activeList', () => {
    it('returns only items with status active', () => { /* ... */ })
  })

  describe('function#getList', () => {
    it('returns empty array when no filters applied', () => { /* ... */ })
    it('throws when page is negative', () => { /* ... */ })
  })

  describe('watch#activeList', () => {
    it('calls onChange callback when activeList reference changes', () => { /* ... */ })
  })

  describe('onMounted', () => {
    it('fetches initial list on mount', () => { /* ... */ })
  })
})
```

```ts
// utility module: src/utils/date.js
describe('date.js', () => {
  describe('getStartOfDay', () => {
    it('returns midnight for a mid-day timestamp', () => { /* ... */ })
  })

  describe('isWeekend', () => {
    it('returns true for a Saturday date', () => { /* ... */ })
    it('returns false for a Wednesday date', () => { /* ... */ })
  })
})
```

- Component prefix kinds: `prop#`, `event#`, `computed#`, `slot#`.
- Composable prefix kinds: `computed#`, `function#` (plain/exposed functions), `watch#`, `watchEffect#`.
- Lifecycle hooks (`onMounted`, `onUnmounted`, `onUpdated`, ...) get no `#name` suffix on either kind —
  at most one of each per component/composable.
- Utility modules (no reactivity, just exported functions): no prefix — `describe('<functionName>')`.

## `it()` naming

One scenario per `it`, phrased `<expected outcome> when <condition>` (extends generic.md's "names
describe the scenario") — third person, present tense, no `should`/`test that`:

- **computed / function:** `'returns <value> when <condition>'`, `'returns <value> for <input>'`,
  `'throws <error> when <invalid input>'`
- **watch / watchEffect:** `'<calls|triggers> <effect> when <dep> changes to <value>'`
- **onMounted / onUnmounted:** `'<does X> on mount'` / `'<does X> on unmount'`
- **component props/computed:** `'renders <thing> when <prop/condition>'`
- **component events:** `'emits <event> with <payload> when <interaction>'`
- **errors (any kind):** name the failure mode, not just "handles error" —
  `'rejects when input is empty'`, `'throws when id is missing'`.

## What to test by target kind

- **Composables / utils (plain functions):** call directly, assert return values + thrown errors.
  Aim for 100% — cover every branch and edge input. No mounting needed. Group tests per the Structure
  section above.
- **Components:**
  - **Props →** rendered output / class / attribute bindings.
  - **Events →** `wrapper.emitted('name')` fired with the correct payload on the correct interaction.
  - **Computed →** derived values for representative inputs (incl. edge inputs).
  - Conditional rendering (`v-if`/`v-show`) toggles for each branch.
- **Stores (Pinia/Vuex):** actions (incl. async), getters, mutations in isolation.

## Mocking

- Mock modules at the boundary: `vi.mock('@/api/orders', () => ({ fetchOrders: vi.fn() }))`.
- Mock i18n/router/store deps the component imports; prefer `createTestingPinia` for stores.
- Prefer `shallowMount` (isolate the unit); use `mount` only when child rendering is under test.
- Select elements via stable `data-test` attributes, not text or CSS that churns.

## Async & errors

- `await flushPromises()` after triggering promise chains before asserting.
- For rejected promises, follow [../cases/handling-rejected-promises.md](../cases/handling-rejected-promises.md)
  to avoid "Unhandled Rejection" warnings.
